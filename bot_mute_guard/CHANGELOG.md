# 更新日志

## 1.1.0 (2026-05-21)
- 适配 TelePilot 0.21.0 新增的 `moderate_chat` 受控成员管理能力，模块权限声明新增 `moderate_chat`。
- 新增“违规处理动作”配置，支持仅删除消息、禁言发送者、踢出发送者、封禁发送者；默认仍为仅删除消息，避免升级后自动处理成员。
- 新增“禁言时长（秒）”配置，违规处理动作为禁言时生效，`0` 表示永久禁言。
- 命中非白名单 `@bot`、inline Bot、Bot 发言和可识别 Bot 入群服务消息后，会在删除消息后按配置处理对应违规成员，并写入运行日志。
- 补充冒烟测试，覆盖禁言与踢出动作调用，避免误把白名单/演练模式修复回退。

## 1.0.4 (2026-05-21)
- 修复布尔配置字符串解析问题：当 TelePilot 将关闭状态传为 `"false"` / `"off"` / `"0"` 等字符串时，模块会正确识别为关闭，不再把演练模式误判为开启。
- 同步修复所有布尔开关的解析逻辑，避免群内提示、删除规则等开关在字符串配置下出现反向行为。
- 消息处理前会按 TelePilot 热重载后的最新 `ctx.config` 重新应用配置，白名单、演练模式和目标群保存后无需重启模块即可生效。
- 补充冒烟测试，覆盖 `dry_run = "false"` 必须真实删除、运行中移除白名单后应立即删除，以及删除规则字符串关闭时不应误触发。

## 1.0.3 (2026-05-21)
- 修复直接由 Bot 发言时事件未预填 `sender` 导致无法识别的问题：在目标群内会使用 `event.get_sender()` 读取发送者并按 Bot ID / username 判断白名单。
- 改进目标群 `@username` 匹配：当事件未预填 `chat` 时会使用 `event.get_chat()` 补充读取，减少配置为群 username 时静默不触发的问题。
- 权限声明新增 `read_chat`，覆盖读取发送者/聊天上下文的只读能力；同步更新白名单说明，区分文本 @bot、inline via_bot_id 与直接 Bot 发言的判断方式。

## 1.0.2 (2026-05-21)
- 优化 Bot 白名单配置说明：明确推荐每行填写一个 Bot username，`@` 可写可不写，大小写不敏感。
- 同步更新运行日志版本号与远程元数据，便于确认 TelePilot 已加载新版本。

## 1.0.1 (2026-05-21)
- 按 TelePilot 0.19.2 远程模块开发文档严格兼容：改用 `min_telepilot_version`、`category = "automation"`、`x-ui-mode = "single"`，并同步补充兼容展示用的 `x-category`，保持 `plugin.json` 与 `manifest.py` 元数据一致。
- 移除远程模块沙箱未授权的成员管理路径，不再调用 `get_entity`、`edit_permissions` 或 raw MTProto；模块只使用已声明的 `delete_message` 与 `send_message` 能力。
- 将非白名单 `@xxxbot`、inline Bot 与可识别 Bot 发言的处理收敛为删除违规消息、写运行日志和可选群内提示，避免越过 TelePilot 权限边界导致运行失败。
- 更新冒烟测试，覆盖非白名单 `@dddbot` 删除、白名单 `/命令@qqqbot` 忽略、纯 `@defbot` 删除以及邮箱样式不误删。

## 1.0.0 (2026-05-21)
- 新增 Bot 防广告守卫插件草案：支持目标群、Bot 白名单、inline Bot 处理与演练模式配置。
