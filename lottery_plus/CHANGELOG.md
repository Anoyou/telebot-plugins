# 更新日志
## 1.0.7 (2026-06-19)
- 按 TelePilot 最新交互 Bot 入口规范补齐 `launch_mode`、事件白名单、会话策略、payload/result contract 和结算声明。
- 保留原有 UserBot 命令触发，交互 Bot 入口只负责触发和高频互动承接，不改变插件本体配置。
- 按最新插件开发指南补充 `min_telepilot_version`，并将配置页模式从旧兼容别名 `schema` 更新为推荐的 `single`。

## 1.0.6 (2026-06-19)
- 修复购买彩票时记录的玩家展示名可能读取账号本地联系人备注的问题；保存为联系人时优先展示公开 username（不带 @），没有 username 时回退用户 ID。
- 兼容新版 TelePilot 的统一公开展示名 helper，并保留旧环境兜底逻辑。

## 1.0.5 (2026-05-22)
- 补充远程模块配置页使用说明：提供彩票模块真实命令、查询入口与交互 Bot 下注说明。
- 使用 `{prefix}` / `{command}` 占位符生成说明，兼容系统命令前缀变更。

## 1.0.4 (2026-05-21)
- 修复消息预览中的指令前缀占位符：预览默认值保留 `{prefix}`，由前端按当前系统前缀渲染，不再固定显示英文逗号。
- 修复 `plugin.json` 中模板默认值的换行转义，避免配置页把 `\n` 当普通文本展示。
- 将配置页模式从旧兼容别名 `schema` 调整为最新推荐的 `single`。

## 1.0.3 (2026-05-21)
- 修复交互 Bot 启动彩票模块后只重复“发送命令”提示的问题；现在正常用户转账只会按金额判定为下注，不再进入短会话式参与流程。
- 交互 Bot 玩法提示改为说明转账金额编码：`基础下注金额 × 注数 + 号码`。
- 彩票帮助与交互查询提示改为使用 `{prefix}` / 当前 userbot 全局指令前缀，不再硬编码逗号。
- 开奖周期改回每日固定时间模型，默认每日 21:00 开奖、开奖前 1 分钟封盘；查询仍使用彩票模块自己的命令入口。

## 1.0.2 (2026-05-21)
- 兼容最新开发文档：在 `plugin.json` 中补齐 `config_schema`，修复配置页不显示配置项的问题。
- 增加 `min_telepilot_version` 字段，并保留 `min_telebot_version` 作为兼容别名。
- 同步版本号到 `manifest.py` 与 `plugin.json`。

## 1.0.1 (2026-05-20)
- 补充交互 Bot 运行入口：实现 `on_interaction(ctx, entry_key, payload)` 最小 hook。
- 保持原有指令与消息监听主逻辑不变，仅新增交互入口声明对应的标准动作返回。
- 同步更新版本号与远程元数据一致性。

## 1.0.0

- 提供管理员指令（强制开奖、清盘、赞助增减奖池、退款）。

- 增加配置页只读消息预览，便于上线前校对文案。
