# 更新日志

## 1.2.2 (2026-07-02)
- 在插件启动日志和 `help` 输出中展示当前运行版本，方便确认远程更新后 worker 是否真正加载了新代码。

## 1.2.1 (2026-07-02)
- 修复 `event_subscriptions` 中消息事件 scope 使用非法值 `chat` 的规范问题，改为开发指南允许的 `all_allowed_chats`。

## 1.2.0 (2026-07-02)
- 去除同一聊天只能存在一个红包的限制，现在可同时发多个拼手气红包。
- 新增红包代码，红包消息、列表和结算都会展示对应代码。
- 新增 `list` 参数查看当前聊天红包列表，新增 `off 红包代码` 强制关闭指定红包。
- 图片财富密码失败时会写入插件日志并给出明确错误，方便定位 Pillow 依赖或发送能力问题。
- 补齐配置热更新键，图片开关、删除原指令、默认金额等配置调整后能被运行态重新读取。

## 1.1.0 (2026-07-02)
- 修复开启“发出后删除原指令”后，部分命令事件没有 `delete()` 能力导致原指令未删除的问题。
- 删除原指令现在会在事件删除不可用时，改用 UserBot client 按消息 ID 删除。
- 领取后不再编辑旧红包消息，改为删除旧消息并重发最新红包内容，让更新后的财富密码回到群聊底部。
- 当前红包内容新增可折叠领取详情，避免领取记录长期占用屏幕。
- 新增图片财富密码模式，可通过配置默认开启，也可用 `img` / `text` 指令参数临时切换。
- 图片财富密码参考旧红包插件的验证码干扰逻辑生成，红包正文不再暴露可复制的财富密码文本。

## 1.0.1 (2026-07-02)
- 调整红包消息模板，财富密码行只展示可领取口令，不再附带括号说明。
- 将领取提示拆成独立一行，文案改为“提示：财富密码被领一次会随机变动”。

## 1.0.0 (2026-07-02)
- 新增独立拼手气口令红包插件，支持通过可配置指令创建红包。
- 财富密码由基础口令追加随机码组成，例如 `发财A7K9`，每次领取后自动刷新。
- 领取奖励固定由 UserBot 回复领取者消息发送 `+金额`，不提供交互 Bot 入口。
- 支持默认总额、默认人数、最低单包、随机码位数、有效期和发起人是否可领取等配置。
